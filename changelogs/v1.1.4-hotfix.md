## 🔧 文档修复

### 1. 集合依赖自动失效功能说明修正

**问题**: v1.1.4 文档声称支持集合依赖自动失效功能，但代码中未实现

**影响文件**:
- `docs/function-cache.md`
- `changelogs/v1.1.4.md`

**修复内容**:
- ❌ 移除了关于"MongoDB 写操作会自动失效相关函数缓存"的误导性描述
- ✅ 添加了正确的使用说明：数据更新后需要手动失效缓存
- ✅ 提供了正确的手动失效示例代码

**修复前**（错误）:
```javascript
// 当 MongoDB 集合更新时，相关函数缓存自动失效
await msq.collection('users').updateOne({ _id: 'user123' }, { $set: { name: 'Alice' } });
// ✅ getUserProfile 的缓存已自动失效
```

**修复后**（正确）:
```javascript
// ⚠️ 重要：数据更新后需要手动失效缓存
await msq.collection('users').updateOne({ _id: 'user123' }, { $set: { name: 'Alice' } });
await fnCache.invalidate('getUserProfile', 'user123'); // 手动失效缓存
```

**技术说明**:
- `_registerDependencies` 方法目前只存储依赖关系，不执行实际的自动失效逻辑
- `collections` 参数可以保留（用于文档记录），但不会触发自动失效
- 用户必须在数据更新后手动调用 `invalidate()` 方法

## 🎯 用户行动建议

### 如果你正在使用 v1.1.4

**❌ 不要依赖自动失效功能**

如果你的代码依赖于集合依赖自动失效功能，请立即修改：

```javascript
// ❌ 错误：假设会自动失效
fnCache.register('getUserProfile', getUserProfileFn, {
    collections: ['users']  // 这不会触发自动失效
});
await msq.collection('users').updateOne(...);
// 缓存不会自动失效！

// ✅ 正确：手动失效
fnCache.register('getUserProfile', getUserProfileFn, {
    ttl: 300000
});
await msq.collection('users').updateOne(...);
await fnCache.invalidate('getUserProfile', userId); // 手动失效
```

**升级操作**:
1. 检查你的代码中是否使用了 `collections` 参数
2. 在所有数据更新操作后添加 `invalidate()` 调用
3. 测试缓存失效逻辑是否正常工作

## 📝 技术债务

### 待实现功能（v1.2.0 计划）

**集合依赖自动失效功能**

如果社区有需求，可以在未来版本实现：

1. **实现方案**:
   ```javascript
   // lib/mongodb/collection.js
   async updateOne(filter, update, options) {
       const result = await this._adapter.updateOne(filter, update, options);
       
       // 自动失效函数缓存
       if (this._msq._functionCache) {
           await this._msq._functionCache._invalidateByCollection(this.name);
       }
       
       return result;
   }
   
   // lib/function-cache.js
   async _invalidateByCollection(collectionName) {
       const depsKey = `fn_deps:${collectionName}`;
       const functions = await this.cache.get(depsKey);
       
       if (functions && Array.isArray(functions)) {
           for (const fnName of functions) {
               await this.invalidatePattern(`${fnName}:*`);
           }
       }
   }
   ```

2. **工作量**: 中等（需要在所有写操作中集成 Hook）

3. **风险评估**:
   - 性能影响：每次写操作都需要检查依赖关系
   - 可靠性：需要确保所有写操作都触发失效
   - 复杂性：需要维护完整的依赖关系图

4. **决策**: 暂不实现，优先保持简单性
