### 新增功能 ✨

#### 1. 虚拟字段（Virtuals）- 计算属性

**功能说明**:
- 计算属性，不存储在数据库中
- getter 函数：定义计算逻辑（如 firstName + lastName = fullName）
- setter 函数：反向赋值支持（可选）
- 自动注入：查询结果自动应用虚拟字段
- JSON 序列化：虚拟字段在 JSON.stringify 中可见

**使用示例**:
```javascript
Model.define('users', {
    schema: (dsl) => dsl({
        firstName: 'string!',
        lastName: 'string!'
    }),
    virtuals: {
        fullName: {
            get: function() {
                return `${this.firstName} ${this.lastName}`;
            },
            set: function(value) {
                const parts = value.split(' ');
                this.firstName = parts[0];
                this.lastName = parts[1];
            }
        }
    }
});

// 使用
const user = await User.findOne({ firstName: 'John' });
console.log(user.fullName); // 'John Doe'（虚拟字段）

// setter 支持
user.fullName = 'Jane Smith';
console.log(user.firstName); // 'Jane'
console.log(user.lastName);  // 'Smith'
```

#### 2. 默认值（Defaults）- 插入时自动填充

**功能说明**:
- 静态默认值：如 status: 'active'
- 函数默认值：如 createdAt: () => new Date()
- 上下文默认值：如 createdBy: (ctx) => ctx.userId
- 自动应用：insertOne/insertMany 时自动填充
- 不覆盖已提供的值

**使用示例**:
```javascript
Model.define('users', {
    schema: (dsl) => dsl({
        username: 'string!',
        status: 'string?',
        score: 'number?',
        createdAt: 'date?'
    }),
    defaults: {
        status: 'active',              // 静态值
        score: 0,                      // 静态值
        createdAt: () => new Date()    // 函数值
    }
});

// 使用
await User.insertOne({ username: 'john' });
// 自动应用默认值：
// status = 'active'
// score = 0
// createdAt = 当前时间

// 不覆盖已提供的值
await User.insertOne({ 
    username: 'alice',
    status: 'inactive'  // 保留用户提供的值
});
// status = 'inactive'（不会被默认值覆盖）
```

### Bug 修复 🐛

#### 嵌套 Populate + Select 问题修复 ✅

**问题描述**:
- 嵌套 populate 配置中使用 select 选项时，关联数据未填充
- 表现：二级及以上的关联数据为空或 undefined

**根本原因**:
- select 选项会过滤掉用于构建关系映射的外键字段
- 导致 populate 逻辑无法找到关联数据

**修复方案**:
- 在 `_selectFields` 方法中保留外键字段（keepField 参数）
- 确保外键字段在内部映射时可用，但不暴露给用户

**技术细节**:
```javascript
// 修复前：select 会过滤掉外键字段
_selectFields(doc, select) {
    const result = { _id: doc._id };
    select.forEach(field => {
        if (doc[field] !== undefined) {
            result[field] = doc[field];
        }
    });
    // ❌ 外键字段被过滤，导致映射失败
    return result;
}

// 修复后：保留外键字段用于内部映射
_selectFields(doc, select, keepField) {
    const result = { _id: doc._id };
    select.forEach(field => {
        if (doc[field] !== undefined) {
            result[field] = doc[field];
        }
    });
    // ✅ 保留外键字段，确保关系映射正确
    if (keepField && doc[keepField] !== undefined) {
        result[keepField] = doc[keepField];
    }
    return result;
}
```

**影响范围**:
- 仅影响嵌套 populate + select 的组合使用
- 不影响单层 populate 或不使用 select 的场景

**受影响的使用场景**:
```javascript
// ❌ v1.0.3 会失败（comments 为空）
const user = await User.findOne({ _id }).populate({
    path: 'posts',
    populate: {
        path: 'comments',
        select: 'content'  // select 会导致问题
    }
});

// ✅ v1.0.4 已修复
// comments 数组正确填充，且只包含 _id 和 content 字段
```

**详细报告**: nested-populate-bugfix-v1.3.0.md（历史路径: `../reports/monSQLize/patches/nested-populate-bugfix-v1.3.0.md`）

## TypeScript 支持完善 ✅

### 类型定义改进

**Relations 类型修复**:
```typescript
// ❌ 修复前：ORM 风格（不符合 MongoDB 原生）
interface RelationConfig {
    type: 'hasOne' | 'hasMany' | 'belongsTo';
    model: string;
    foreignKey: string;
}

// ✅ 修复后：MongoDB 原生风格
interface RelationConfig {
    type: 'one' | 'many';
    ref: string;
    localField: string;
    foreignField: string;
}
```

**新增类型**:
```typescript
// 虚拟字段配置
interface VirtualConfig {
    get: (this: any) => any;
    set?: (this: any, value: any) => void;
}

// 默认值类型
type DefaultValue<T> = T | (() => T) | ((ctx: any) => T);

// Populate 代理接口
interface PopulateProxy<T = any> {
    populate(path: string | PopulateConfig): PopulateProxy<T>;
    exec(): Promise<T>;
}

// Populate 配置接口
interface PopulateConfig {
    path: string;
    select?: string | string[];
    match?: any;
    sort?: any;
    limit?: number;
    skip?: number;
    populate?: string | PopulateConfig | PopulateConfig[];
}
```

**ModelInstance 方法返回类型更新** (6个):
- `find()` → `PopulateProxy<T[]>`
- `findOne()` → `PopulateProxy<T | null>`
- `findById()` → `PopulateProxy<T | null>`
- `findByIds()` → `PopulateProxy<T[]>`
- `findPage()` → `PopulateProxy<PageResult<T>>`
- `findAndCount()` → `PopulateProxy<{ rows: T[], count: number }>`

```bash
npm install monsqlize@1.0.4
```

### 使用虚拟字段

虚拟字段适用于：
- ✅ 计算属性（如 fullName = firstName + lastName）
- ✅ 数据格式化（如日期格式化、货币格式化）
- ✅ 业务逻辑封装（如订单状态描述）

**不适用于**:
- ❌ 需要在数据库查询中使用的字段
- ❌ 需要排序/筛选的字段

### 使用默认值

默认值适用于：
- ✅ 状态字段（如 status: 'active'）
- ✅ 计数器字段（如 score: 0）
- ✅ 时间戳字段（如 createdAt: () => new Date()）

**不适用于**:
- ❌ 复杂的业务逻辑（使用 hooks 更合适）
- ❌ 依赖其他字段的计算（使用 hooks 更合适）
