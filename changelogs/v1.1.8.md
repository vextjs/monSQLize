## 🆕 新功能

### ✨ Model.undefine(collectionName) — 注销已注册的 Model 定义

**功能概述**:
- 从全局注册表中移除指定的 Model 定义
- 返回 `boolean`（`true` = 成功移除，`false` = 不存在）
- 已实例化的 `ModelInstance` 不受影响
- 幂等操作，对不存在的 model 不抛错
- 主要用途：开发模式下的 Model 热重载

**使用示例**:
```javascript
const { Model } = require('monsqlize');

// 注销 Model 定义
const removed = Model.undefine('users');  // true
Model.undefine('nonexistent');            // false（不抛错）

// 热重载场景
Model.undefine('users');
Model.define('users', newDefinition);
```

**API 签名**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `collectionName` | `string` | ✅ | 要注销的 Model 集合名称 |

| 返回值 | 类型 | 说明 |
|--------|------|------|
| `result` | `boolean` | `true` = 成功移除，`false` = 不存在 |

**行为特性**:
- ✅ 幂等操作：多次调用不抛错
- ✅ 已实例化的 `ModelInstance` 不受影响（引用保持有效）
- ✅ 注销后可重新 `define()` 同名 Model
- ❌ 不会影响已建立的数据库连接

---

### ✨ Model.redefine(collectionName, definition) — 重新定义已注册的 Model

**功能概述**:
- 等效于 `undefine()` + `define()` 的组合操作
- 如果 Model 不存在，行为等同于 `define()`
- ⚠️ 注意：如果新定义校验失败，旧定义会被移除
- 主要用途：一步完成 Model 热重载

**使用示例**:
```javascript
const { Model } = require('monsqlize');

// 一步替换
Model.redefine('users', {
    schema: (dsl) => dsl({ username: 'string!', email: 'string!' })
});

// 首次定义（等同于 define()）
Model.redefine('newCollection', {
    schema: (dsl) => dsl({ name: 'string!' })
});

// 多轮重载
Model.redefine('users', definitionV1);
Model.redefine('users', definitionV2);
Model.redefine('users', definitionV3);  // 每次都替换为最新定义
```

**API 签名**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `collectionName` | `string` | ✅ | Model 集合名称 |
| `definition` | `object` | ✅ | 新的 Model 定义对象 |

**行为特性**:
- ✅ 已存在的 Model → 先移除再重新定义
- ✅ 不存在的 Model → 直接定义（等同于 `define()`）
- ⚠️ 新定义校验失败时，旧定义**已被移除**（不会回滚）
- ✅ 已实例化的 `ModelInstance` 不受影响

### ✨ _loadModels() 增加 reload 模式

**功能概述**:
- 方法签名新增 `options` 参数：`_loadModels({ reload: true })`
- `reload=true` 时，已存在的 model 使用 `redefine()` 替换
- `reload=false`（默认）保持原有行为：已注册则跳过
- `connect()` 内部调用不变（默认不 reload），完全向后兼容

**使用示例**:
```javascript
const MonSQLize = require('monsqlize');

const msq = new MonSQLize({
    type: 'mongodb',
    databaseName: 'myapp',
    config: { uri: 'mongodb://localhost:27017' }
});

// 默认行为（不 reload）— 已注册的 model 跳过
await msq._loadModels();

// 热重载模式 — 已注册的 model 使用 redefine() 替换
await msq._loadModels({ reload: true });
```

**参数说明**:

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `options` | `object` | `{}` | 可选配置对象 |
| `options.reload` | `boolean` | `false` | 是否重新加载已存在的 model |

**行为对比**:

| 场景 | `reload: false`（默认） | `reload: true` |
|------|------------------------|----------------|
| Model 不存在 | `define()` 注册 | `define()` 注册 |
| Model 已存在 | 跳过（不操作） | `redefine()` 替换 |
| `connect()` 调用 | 使用默认值 | 不变（默认不 reload） |

## 🔧 改进

### TypeScript 类型声明更新

- 修正 Model namespace 中的 `remove` → `undefine`
- 新增 `redefine` 方法签名
- 新增 `RegisteredModel<T>` 接口，修正 `Model.get()` 返回值类型（原声明为裸 `ModelDefinition<T>`，实际返回 `{ collectionName, definition }` 包装对象）（v1.1.8 修正）
- 移除类型声明中不存在的 `clear()` 方法（实现为私有方法 `_clear()`，不应暴露在公共类型中）（v1.1.8 修正）
- 完整的类型提示支持

**类型定义变更**:
```typescript
// 🆕 新增 — Model.get() 返回值结构
export interface RegisteredModel<T = any> {
  collectionName: string;
  definition: ModelDefinition<T>;
}

declare namespace Model {
    // 🆕 新增
    function undefine(collectionName: string): boolean;
    function redefine(collectionName: string, definition: ModelDefinition): void;

    // ✏️ 修正返回类型（原为 ModelDefinition<T> | undefined）
    function get<T = any>(collectionName: string): RegisteredModel<T> | undefined;

    // ❌ 移除（私有方法 _clear() 不应出现在公共类型中）
    // function clear(): void;

    // 已有（不变）
    function define(collectionName: string, definition: ModelDefinition): void;
}
```
