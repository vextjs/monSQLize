## Bug Fix：`model()` 方法使用注册 key 作为 MongoDB 集合名

### 问题

`model(collectionName)` 获取 collection 实例时，直接将传入的注册 key（`collectionName`）作为 MongoDB 集合名传给 `_resolveModelCollection` 和 `dbInstance.collection()`。

当上层框架（如 vext）使用带路径前缀的注册 key（如 `AOrder`，由目录 `a/order.ts` 推断），而 `definition.name` 存储了实际 MongoDB 集合名（如 `order`）时，MongoDB 访问的集合名与预期不符。

### 根因

```js
// v1.2.2 — 直接使用注册 key 作为集合名
const collection = (connection && (connection.pool || connection.database))
  ? this._resolveModelCollection(collectionName, connection)
  : this.dbInstance.collection(collectionName);
```

### 修复

在获取 collection 前，优先从 `definition.collection` 或 `definition.name` 读取实际集合名，注册 key 仅作为 fallback：

```js
// [v1.2.3] 实际 MongoDB 集合名可能不同于注册 key
const actualCollectionName =
  modelDef.definition.collection ||
  modelDef.definition.name ||
  collectionName;  // fallback：注册 key（向后兼容）

const collection = (connection && (connection.pool || connection.database))
  ? this._resolveModelCollection(actualCollectionName, connection)
  : this.dbInstance.collection(actualCollectionName);
```

### 兼容性

- 现有 model（注册 key 与 MongoDB 集合名相同，如直接传 `"admin_login_logs"`）：
  `actualCollectionName = definition.collection || definition.name || "admin_login_logs"` — 行为完全不变 ✅
- 新场景（框架注入 `definition.name` 实际集合名）：正确使用 `definition.name` ✅
