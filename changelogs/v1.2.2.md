## 新功能：Model 数据源绑定

### 背景

v1.0.8 引入了多连接池（`ConnectionPoolManager`），但 `msq.model()` 始终固定走默认连接池。
在一个连接池对应多个数据库、或多连接池场景下，无法将特定 Model 路由到指定的连接池 + 数据库组合。

### 解决方案

在 `Model.define()` 的 `definition` 对象中新增可选字段 `connection`：

```js
Model.define('AnalyticsReport', {
  schema: (dsl) => dsl({
    reportId: 'string!',
    generatedAt: 'date'
  }),
  connection: {
    pool: 'analytics',      // 可选：连接池名称（须与 pools[].name 对应）
    database: 'reports_db'  // 可选：数据库名称（不填则使用实例 databaseName）
  }
});
```

调用 `msq.model('AnalyticsReport')` 时自动路由到对应的连接池 + 数据库。

---

## 四种路由行为

| `pool` | `database` | 路由路径 |
|:------:|:----------:|---------|
| 未配置 | 未配置 | `dbInstance.collection(name)` — **原逻辑，向后兼容** |
| 未配置 | `'logs_db'` | `dbInstance.db('logs_db').collection(name)` |
| `'analytics'` | 未配置 | 指定池 + `this.databaseName` |
| `'analytics'` | `'reports_db'` | 指定池 + 指定数据库 |

---

## 错误码

| 错误码 | 触发条件 | 说明 |
|--------|---------|------|
| `INVALID_MODEL_DEFINITION` | `connection.pool` 或 `connection.database` 为空字符串/非字符串 | `Model.define()` 阶段抛出 |
| `NO_POOL_MANAGER` | 配置了 `connection.pool` 但实例未配置 `pools` | `msq.model()` 调用时抛出 |
| `POOL_NOT_FOUND` | 指定的池名在 `poolManager` 中不存在 | `msq.model()` 调用时抛出，错误信息含可用池列表 |

## 使用示例

```js
const msq = new MonSQLize({
  uri: 'mongodb://localhost:27017',
  databaseName: 'main_db',
  pools: [
    { name: 'analytics', uri: 'mongodb://analytics-host:27017' }
  ]
});

// 场景 1：Model 使用不同数据库（默认连接池）
Model.define('AuditLog', {
  schema: (dsl) => dsl({ action: 'string!', userId: 'string!' }),
  connection: { database: 'audit_db' }
});

// 场景 2：Model 使用不同连接池（使用实例默认数据库名）
Model.define('AnalyticsEvent', {
  schema: (dsl) => dsl({ eventType: 'string!', payload: 'object' }),
  connection: { pool: 'analytics' }
});

// 场景 3：Model 使用不同连接池 + 不同数据库
Model.define('AnalyticsReport', {
  schema: (dsl) => dsl({ reportId: 'string!', data: 'object' }),
  connection: { pool: 'analytics', database: 'reports_db' }
});

await msq.connect();

// 正常使用，路由自动处理
const AuditLogModel  = msq.model('AuditLog');      // → audit_db（默认池）
const EventModel     = msq.model('AnalyticsEvent'); // → main_db（analytics 池）
const ReportModel    = msq.model('AnalyticsReport');// → reports_db（analytics 池）
const UserModel      = msq.model('User');           // → main_db（默认池，兼容原逻辑）
```

## Bug Fix：Model._clear() 导致 msq.model() 缓存失效失效

**问题**：`Model._clear()` 在清空注册表（`_registry`）的同时，也清空了 `_redefinedNames`。
`msq.model()` 通过检查 `_redefinedNames` 来判断缓存是否需要重建，清空后缓存永远不会失效，
导致 `_clear()` 后重新 `define()` 的 Model 仍复用旧的 `ModelInstance`，旧实例的 `_relations` 不含新定义的关系，所有 `populate()` 调用均抛出 "未定义的关系: xxx" 错误。

**修复**：`_clear()` 先将所有已注册名加入 `_redefinedNames`，再清空 `_registry`，不清空 `_redefinedNames`，确保 `msq.model()` 必然重建 `ModelInstance`。

**影响**：修复后 model 套件 445/445 通过（修复前 439 passing 6 failing）。
