## 🎉 新增功能

### 函数缓存 (Function Cache)

新增通用函数缓存功能，允许为任意异步函数添加缓存能力，不仅限于数据库查询。

#### 核心特性

- ✅ **零侵入装饰器模式** - 通过 `withCache()` 包装函数，不修改原代码
- ✅ **FunctionCache 类** - 统一管理多个函数的缓存
- ✅ **自动参数序列化** - 支持对象、数组、Date、ObjectId 等复杂类型
- ✅ **TTL 过期控制** - 灵活的缓存时间设置
- ✅ **自定义键生成器** - 支持自定义缓存键生成逻辑
- ✅ **命名空间隔离** - 多模块缓存互不干扰
- ✅ **条件缓存** - 基于返回值决定是否缓存
- ✅ **并发控制** - 防止缓存击穿，多个并发请求共享结果
- ✅ **统计监控** - 命中率、调用次数、平均耗时等
- ✅ **完整 Redis 支持** - 仅 Redis / 本地 + Redis 双层缓存
- ✅ **错误容错** - 缓存读写失败不影响业务逻辑

#### API 新增

```javascript
const { withCache, FunctionCache } = require('monsqlize');

// 方式 1: 装饰器
const cached = withCache(fn, { 
    ttl: 60000, 
    cache: msq.getCache()
});

// 方式 2: FunctionCache 类
const fnCache = new FunctionCache(msq);
fnCache.register('myFunction', fn);
```

#### 集合依赖自动失效 🆕

当声明 `collections` 选项时，MongoDB 写操作会自动失效相关函数缓存：

```javascript
fnCache.register('getUserProfile', getUserProfileFn, {
    ttl: 300000,
    collections: ['users', 'orders']  // 声明依赖
});

// 数据更新时自动失效
await msq.collection('users').updateOne({ _id: 'user123' }, { $set: { name: 'Alice' } });
// ✅ getUserProfile 的缓存已自动失效
```

**支持的写操作**:
- updateOne, updateMany
- deleteOne, deleteMany
- insertOne, insertMany
- replaceOne, findOneAndUpdate, findOneAndDelete, findOneAndReplace

## 🔧 Bug 修复

### MultiLevelCache.delPattern 增强

**问题**: 在多层缓存（本地 + Redis）场景下，`delPattern()` 只删除本地缓存，不删除远端 Redis 缓存，导致缓存失效不完整。

**修复**: 
- `MultiLevelCache.delPattern()` 现在会同时删除本地和远端缓存
- 确保 `FunctionCache.invalidate()` 在双层缓存下正确工作

**影响范围**: 
- 使用多层缓存的用户
- 使用缓存失效功能的场景

## 📋 使用示例

### 示例 1: 基础用法

```javascript
const { withCache } = require('monsqlize');

// 复杂业务函数
async function getUserProfile(userId) {
    const user = await msq.collection('users').findOne({ _id: userId });
    const orders = await msq.collection('orders').find({ userId }).toArray();
    return { user, orders };
}

// 添加缓存
const cachedGetUserProfile = withCache(getUserProfile, {
    ttl: 300000,
    cache: msq.getCache()
});

// 使用
await cachedGetUserProfile('user123');  // 首次：从数据库查询
await cachedGetUserProfile('user123');  // 再次：从缓存读取 ⚡
```

### 示例 2: 集合依赖自动失效 🆕

```javascript
const { FunctionCache } = require('monsqlize');

const fnCache = new FunctionCache(msq);

// 注册时声明依赖
fnCache.register('getUserProfile', getUserProfileFn, {
    ttl: 300000,
    collections: ['users', 'orders']  // 🆕 声明依赖
});

// 使用
await fnCache.execute('getUserProfile', 'user123');

// 更新数据
await msq.collection('users').updateOne({ _id: 'user123' }, { $set: { name: 'Alice' } });
// ✅ getUserProfile 的缓存已自动失效

await fnCache.execute('getUserProfile', 'user123'); // 重新查询
```

### 示例 3: 多函数依赖同一集合

```javascript
// 多个函数依赖 users 集合
fnCache.register('getUserProfile', fn1, { collections: ['users'] });
fnCache.register('getUserStats', fn2, { collections: ['users'] });

// 更新 users 集合
await msq.collection('users').updateOne(...);
// ✅ 两个函数的缓存都自动失效
```

## ⚠️ 注意事项

### Redis 配置

如果使用 Redis 缓存，请使用 `127.0.0.1` 而不是 `localhost`，并使用 `db 0`：

```javascript
// ✅ 推荐
MonSQLize.createRedisCacheAdapter('redis://127.0.0.1:6379/0')

// ❌ 可能有问题（在某些 Redis 配置下）
MonSQLize.createRedisCacheAdapter('redis://localhost:6379/1')
```

### 缓存键生成

函数缓存会自动序列化参数生成缓存键：
- 对象键会自动排序（确保 `{a:1, b:2}` 和 `{b:2, a:1}` 生成相同的键）
- 支持 Date、RegExp、ObjectId 等特殊类型
- `undefined` 序列化为空，`null` 序列化为 `null`

### 集合依赖

- `collections` 选项是可选的
- 不指定则依赖 TTL 自然过期或手动失效
- 支持声明多个集合：`collections: ['users', 'orders', 'products']`
