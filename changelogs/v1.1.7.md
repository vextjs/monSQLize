## 📝 变更摘要

为 monSQLize 添加 Model 热重载支持：新增 `Model.undefine()` / `Model.redefine()` 静态方法，`_loadModels()` 支持 reload 模式。

> ⚠️ 注意：v1.1.7 的功能实际随 v1.1.8 版本发布（package.json version 为 1.1.8）。
> 详细 API 文档、测试覆盖矩阵和升级指南请参见 `changelogs/v1.1.8.md`。

---

## 🆕 新功能

### ✨ Model.undefine(collectionName)

从全局注册表中移除指定的 Model 定义。

- 返回 `boolean`（true = 成功移除，false = 不存在）
- 幂等操作，对不存在的 model 不抛错
- 已实例化的 ModelInstance 不受影响

### ✨ Model.redefine(collectionName, definition)

一步完成 Model 定义替换（等效于 undefine + define）。

- 如果 Model 不存在，行为等同于 `define()`
- ⚠️ 新定义校验失败时，旧定义会被移除
- 主要用于开发模式下的 Model 热重载

### ✨ _loadModels() 增加 reload 模式

- 新增 `options.reload` 参数
- `reload=true` 时，已存在的 model 使用 `redefine()` 替换
- `reload=false`（默认）保持原有行为
- `connect()` 内部调用不变，完全向后兼容
