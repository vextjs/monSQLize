## 📝 变更摘要

修复 `findPage` 静默忽略 `projection` 参数的问题，正式支持字段投影过滤，并引入"有效投影"策略自动保护排序字段以确保游标正确生成。

---

## 🎯 背景说明

### 现状问题

`findPage` 使用 MongoDB 聚合管道（`collection.aggregate`）执行查询，而 `buildPagePipelineA` 构建的管道中从未包含 `$project` 阶段：

- `FindPageOptions extends FindOptions` 在 TypeScript 类型层面继承了 `projection`，给用户带来虚假承诺
- 运行时 `find-page.js` 从未读取 `options.projection`，传入的投影被**完全静默忽略**
- 文档参数表缺少 `projection` 条目，测试零覆盖

### 关键约束

`findPage` 通过 `pickAnchor(doc, stableSort)` 从结果文档中提取排序字段值生成游标锚点。若直接透传用户投影而删除排序字段，游标值将变为 `undefined`，导致分页断裂。

### 目标

- 正确实现 `projection` 支持，不破坏任何现有行为
- 自动保护排序字段，确保游标生成始终正确

---

## 📦 变更内容

### 核心变更

引入**有效投影（Effective Projection）**策略：

| 投影类型 | 检测方式 | 处理规则 |
|---------|---------|---------|
| **包含型** `{ field: 1 }` | 无非 `_id` 字段值为 `0`/`false` | 强制追加所有排序字段（值为 `1`） |
| **排除型** `{ field: 0 }` | 含非 `_id` 字段值为 `0`/`false` | 从排除列表中移除排序字段 |
| **数组型** `['f1', 'f2']` | 经 `normalizeProjection` 转换 | 转为包含型后按包含型规则处理 |

`$project` 阶段插入位置：`lookupPipeline` 之后、`before` 方向 `$sort` 恢复之前（不影响排序逻辑）。

### 技术实现

新增 `buildEffectiveProjection(projection, sort)` 辅助函数（`lib/mongodb/queries/find-page.js`），封装上述保护逻辑。`buildPagePipelineA` 新增 `projection` 参数，由调用方（`find-page.js`）计算有效投影后传入。

**三条执行路径均已覆盖**：
1. **游标分页 / 首页**（`doFindPageOne` 普通模式）：使用有效投影
2. **流式模式**（`doFindPageOne` stream 模式）：使用原始 `normalizedProjection`（流无游标提取需求）
3. **Offset-jump 跳页**：手动构建管道，尾部追加有效投影

## 📚 使用示例

```javascript
// 包含型投影（只返回指定字段）
const result = await collection('orders').findPage({
  query: { status: 'paid' },
  sort: { createdAt: -1 },
  limit: 20,
  projection: { orderId: 1, amount: 1 }  // 🆕 新增支持
  // 注：排序字段 createdAt 和 _id 会被自动保留（游标需要）
});

// 数组形式
const result2 = await collection('orders').findPage({
  query: {},
  sort: { createdAt: -1 },
  limit: 20,
  projection: ['orderId', 'status', 'amount']  // 🆕 数组形式也支持
});

// 排除型投影（排除敏感字段）
const result3 = await collection('users').findPage({
  query: {},
  sort: { createdAt: -1 },
  limit: 20,
  projection: { password: 0, token: 0 }  // 🆕 排除型也支持
});
```

**注意事项**：
- 排序字段会被自动保留在返回结果中（保障游标正确生成），无需手动包含
- 如需彻底隐藏排序字段，建议在应用层对返回的 `items` 做后处理
