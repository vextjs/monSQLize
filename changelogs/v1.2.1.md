## 📝 变更摘要

修复 `msq.model()` 每次调用都创建新 `ModelInstance` 的性能问题，引入实例缓存机制；删除 `_createIndexes` 方法重复定义；补全 `types/monsqlize.ts` 中缺失的 `model()` / `collection()` 类型声明。

---

## 🐛 Bug 修复

### 1. `msq.model()` 实例缓存

**问题**：`msq.model(collectionName)` 每次调用都 `new ModelInstance()`，导致：
- 每次触发 `setImmediate(_createIndexes)`，向 MongoDB 发送 `createIndexes` 命令
- 每次重复执行 schema 编译、方法代理、hooks 注册、关系管理器初始化

**修复**：在 `MonSQLize` 实例上增加 `_modelInstances` 缓存 Map，同一 `collectionName` 多次调用直接返回已创建的实例。

**缓存失效机制**：
- `Model.redefine()` / `Model.undefine()` → 标记 `_redefinedNames`，下次 `model()` 调用时自动重建
- `msq.close()` → 清空全部缓存

**性能收益**：

| 指标 | 修复前 | 修复后 |
|------|:------:|:------:|
| 100 QPS 下 `createIndexes` 命令/秒 | 100 | 0（仅启动时 1 次） |
| `model()` 调用开销 | 完整构造（schema 编译 + hooks + 关系 + 索引） | Map.get() O(1) |
| 日志噪音 | 每次输出 "Created N index(es)" | 仅首次输出 |

### 2. 删除 `_createIndexes` 重复定义

**问题**：`lib/model/index.js` 中 `_createIndexes` 方法定义了两次（内容完全相同），JavaScript 后定义覆盖前定义，第一处为死代码。

**修复**：删除第一处定义，保留第二处。

## 🔧 类型补全

### `types/monsqlize.ts` 补全 `model()` / `collection()`

`MonSQLize` interface 之前缺少 `model()` 和 `collection()` 方法的类型声明，已补齐：

```typescript
model(collectionName: string): ModelInstance;
collection(collectionName: string): Collection;
```

## ⚠️ 已知限制

- `_redefinedNames` 为全局 Set，多 MonSQLize 实例场景下首个调用 `model()` 的实例消费标记后，其他实例的缓存不会失效
- 当前使用场景（vext 单实例）无实际影响
