## 版本规范

遵循[语义化版本](https://semver.org/lang/zh-CN/)（Semantic Versioning）:

- **MAJOR** (x.0.0) - 不兼容的 API 变更
- **MINOR** (0.x.0) - 向后兼容的新增功能
- **PATCH** (0.0.x) - 向后兼容的问题修复

示例：
- `1.0.0 → 1.0.1` - Bug 修复
- `1.0.0 → 1.1.0` - 新功能
- `1.0.0 → 2.0.0` - 破坏性变更

## 贡献指南

### 如何记录变更

1. 创建详细变更文档
   - 使用 `changelogs/TEMPLATE.md` 作为模板
   - 文件命名：`changelogs/vX.Y.Z.md`
   - 填写完整的变更信息

2. 更新主 CHANGELOG
   - 在"版本概览"表格最上方添加新行
   - 格式：`| 版本 | 日期 | 变更摘要 | [详细](changelogs/vX.Y.Z.md) |`

3. 提交变更
   ```bash
   git add CHANGELOG.md changelogs/vX.Y.Z.md
   git commit -m "docs: 发布 vX.Y.Z"
   ```
