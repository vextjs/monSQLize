# 03 - 代码风格

> 版本: v1.0.0 | 更新: 2026-04-24

## 模块系统

- **主库**: CommonJS (`require`/`module.exports`)，禁止在 `lib/` 中使用 ES Module 语法
- **类型文件**: TypeScript (`.ts`)，仅描述接口，不含实现
- **ESM 包装**: `index.mjs` 只是对 `lib/index.js` 的代理，不含逻辑

## 类与命名

```js
// ✅ 类名：PascalCase
class ConnectionPoolManager { }
class ModelInstance { }

// ✅ 方法/变量：camelCase
async addPool(config) { }
const poolManager = new ConnectionPoolManager();

// ✅ 私有成员：下划线前缀
this._pools = new Map();
this._poolLock = new AsyncLock();

// ✅ 常量：UPPER_SNAKE_CASE
const DEFAULT_POOL_CONFIG = { maxPoolSize: 100 };

// ✅ 错误码：UPPER_SNAKE_CASE（在 errors.js 中定义）
ErrorCodes.INVALID_COLLECTION_NAME
```

## JSDoc 注释规范

所有公开方法必须有完整 JSDoc：

```js
/**
 * 一句话描述
 *
 * @param {string} name - 参数说明
 * @param {Object} [options] - 可选参数
 * @returns {Promise<void>}
 * @throws {Error} 情况描述
 * @since v1.0.8
 * @example
 * await msq.addPool({ name: 'primary', uri: '...' });
 */
async addPool(name, options) { }
```

## 错误处理

```js
// ✅ 有错误码
const err = new Error('描述');
err.code = 'SOME_ERROR_CODE';
throw err;

// ✅ 或使用工厂（推荐）
const { createError, ErrorCodes } = require('./errors');
throw createError(ErrorCodes.INVALID_CONFIG, '描述');

// ✅ 异步错误不静默丢弃
somePromise().catch(err => {
    this.logger.error('[模块名] 操作失败:', err.message);
});
```

## 可选依赖模式

```js
// ✅ 可选依赖懒加载（ioredis 等）
let dep;
try {
    dep = require('optional-package');
} catch (e) {
    // 输出友好安装提示，不 throw
    console.error('请安装: npm install optional-package');
}
```

## 测试规范

- 测试运行器：`node test/run-tests.js`（**不使用** `mocha` CLI）
- 断言库：Node.js 内置 `assert`（**不使用** chai）
- Mock：`sinon`
- 内存 MongoDB：`mongodb-memory-server`（通过 `config.useMemoryServer: true`）

## 版本标注

新增 API 必须在 JSDoc 中标注 `@since vX.Y.Z`，修改已有 API 需更新注释。
