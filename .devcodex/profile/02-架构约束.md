# 02 - 架构约束

> 优先级: 🔴 P0 最高 | 版本: v1.0.0 | 更新: 2026-04-24

## 强制约束

### C1 禁止破坏 API 向后兼容性

monSQLize 是 npm 公开包，**所有公开 API 变更必须向后兼容**。

- ❌ 禁止删除或重命名已发布的公开方法
- ❌ 禁止修改已有方法的参数顺序或必填性
- ❌ 禁止修改已有方法的返回值结构
- ✅ 新增参数必须设置合理默认值
- ✅ 废弃 API 先标记 `@deprecated` 并保留至少一个版本

### C2 禁止将可选功能变为强制依赖

- ❌ 禁止在核心路径中直接 `require('ioredis')`
- ✅ 可选依赖必须 `try/catch` 懒加载并给友好安装提示
- 注：`schema-dsl` 是 `dependencies`，无需懒加载

```js
// ✅ 正确：可选依赖懒加载
let ioredis;
try { ioredis = require('ioredis'); } catch (e) { /* 给提示 */ }
```

### C3 禁止绕过统一错误系统

- ❌ 禁止直接 `throw new Error('...')` 不带错误码
- ✅ 使用 `lib/errors.js` 中的错误工厂
- ✅ 错误码在 `ErrorCodes` 中定义

```js
// ✅ 正确
const { createError, ErrorCodes } = require('./errors');
throw createError(ErrorCodes.INVALID_COLLECTION_NAME, '集合名称无效');
```

> 例外：`lib/lock/errors.js` 的 `LockAcquireError`/`LockTimeoutError` 是历史遗留独立错误类，保持现状。

### C4 禁止在 `lib/mongodb/` 以外直接操作 MongoDB Collection

- ❌ 禁止在 `lib/model/`、`lib/saga/` 等直接调用 MongoDB 原生方法
- ✅ 必须通过 `lib/mongodb/` 层的封装方法（或传入的 collection 封装对象）

### C5 禁止在核心模块使用 `console.log`

- ❌ 禁止在 `lib/` 目录下直接 console 输出
- ✅ 使用 `lib/logger.js` 的 Logger 实例
- 例外：`require` 失败时的 `console.error` 安装提示

### C6 禁止硬编码连接信息

- ❌ 禁止硬编码 MongoDB 连接串、IP、端口、密码
- ✅ 所有连接信息通过构造函数 `config` 参数传入

### C7 禁止修改 MongoDB 原生方法的返回格式

- ❌ 禁止在 `find()`/`findOne()` 等原生方法包装中修改返回结构
- ✅ 新增便利方法（如 `findPage()`）可定义自己的返回格式

## 模块边界

| 模块 | 职责 | 禁止 |
|------|------|------|
| `lib/mongodb/` | 唯一合法的 MongoDB 原生操作入口 | 在此模块外直接调用 MongoDB Collection |
| `lib/model/` | Schema 验证、Hooks、Populate、关系 | 直接操作 MongoDB，bypass collection 封装 |
| `lib/infrastructure/` | 连接池生命周期、健康检查、选择策略 | 包含业务逻辑 |
| `lib/transaction/` | 事务管理 | 直接操作缓存 |
| `lib/errors.js` | 错误码注册中心 | 在其他模块中定义新错误码 |

## 当前数据库支持边界

- ✅ **MongoDB**：完整实现（`lib/connect.js` → `lib/mongodb/`）
- ❌ **MySQL/PostgreSQL**：路线图，`operators.js` 有映射占位，驱动层未实现
- 同一 `MonSQLize` 实例只能绑定一种数据库 `type`
- 多 MongoDB 数据库：通过 `db('dbName').collection()` 访问
- 多连接池（MongoDB）：通过 `pools[]` 配置 + `ConnectionPoolManager`
