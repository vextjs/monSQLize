# 需求概述 — Model 多数据库/连接池绑定

> **项目**: monSQLize
> **版本**: v1.0.0
> **创建日期**: 2026-04-24
> **状态**: CP1 已确认

## 背景

monSQLize v1.0.8 引入了多连接池（`ConnectionPoolManager`），支持配置多个 MongoDB 连接到不同的 host/副本集。每个 MongoClient 可以访问**多个数据库**（通过 `client.db(dbName)`）。

但 Model 层（`Model.define()`）目前没有绑定数据源的概念。`msq.model('users')` 始终通过默认连接（`this.dbInstance`）访问集合，无法路由到指定连接池或指定数据库。

## 目标场景

```
main 连接池（MongoClient → host1:27017）
├── users_db.users
├── users_db.roles
└── orders_db.orders

analytics 连接池（MongoClient → host2:27017）
├── reports_db.reports
└── stats_db.events
```

开发者希望在 `Model.define()` 声明 Model 使用哪个池 + 哪个数据库，后续调用 `msq.model()` 时自动路由。

## 需求点

### R1 — Model 绑定数据源声明

`Model.define()` 的 `definition` 对象支持新字段 `connection`：

```js
Model.define('orders', {
  schema: (dsl) => dsl({ amount: 'number!' }),
  connection: {
    pool: 'main',         // 指定连接池名称（可选）
    database: 'orders_db' // 指定数据库名称（可选）
  }
});
```

### R2 — `msq.model()` 自动路由

调用 `msq.model('orders')` 时，根据 `connection` 配置自动从正确的（池 + 数据库）获取 collection，对调用者透明。

### R3 — 向后兼容

不配置 `connection` 时，行为与现在完全相同，不影响任何已有代码。

### R4 — 错误处理

- `connection.pool` 指定的池不存在：抛出 `POOL_NOT_FOUND` 错误
- 使用 `connection.pool` 但未配置多连接池（`pools` 为空）：抛出 `NO_POOL_MANAGER` 错误
- `connection.database` 为空字符串：在 `Model.define()` 阶段抛出验证错误

## `connection` 四种组合行为

| pool | database | 行为 |
|:----:|:--------:|------|
| ❌ 未配置 | ❌ 未配置 | 与原来完全相同（向后兼容） |
| ❌ 未配置 | ✅ 已配置 | **默认连接池** + 指定数据库 |
| ✅ 已配置 | ❌ 未配置 | 指定连接池 + 实例初始化时的 `databaseName` |
| ✅ 已配置 | ✅ 已配置 | 指定连接池 + 指定数据库 |

> 只指定 `database` 不指定 `pool` = 默认连接池切换到另一个数据库，等效于 `this.dbInstance.db(database).collection(name)`。

## 不在本期范围

- 按读写操作自动选择不同连接池（如读走 secondary、写走 primary）
- `msq.model('name', { pool })` 调用时动态覆盖（预留扩展点）
- 多数据库类型混用（MongoDB + MySQL）

## 验收标准

1. `Model.define()` 配置 `connection.pool` → `msq.model()` 从指定池获取 collection ✅
2. `Model.define()` 配置 `connection.database` → `msq.model()` 从默认连接的指定数据库获取 collection ✅
3. 两者组合正常工作 ✅
4. 不配置 `connection` 时与原来行为完全一致 ✅
5. 指定不存在的池名抛出 `POOL_NOT_FOUND` ✅
6. 全量现有测试通过（无回归）✅
