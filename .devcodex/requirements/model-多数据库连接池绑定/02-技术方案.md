# 技术方案 — Model 多数据库/连接池绑定

> **项目**: monSQLize
> **版本**: v0.0.1 — 2026-04-24 — 初版
> **状态**: CP2 待确认

## 1. 架构分析

### 当前调用链（`msq.model()`）

```
msq.model('users')
  → this.dbInstance.collection('users')         ← lib/connect.js 返回的闭包
    → this._adapter.collection(databaseName, 'users')  ← lib/mongodb/index.js
      → this.client.db(databaseName).collection('users')  ← 原生 MongoClient
      → 包装成 monSQLize collection 对象（含 find/insert 等方法）
```

### `this.dbInstance` 结构（`lib/connect.js`）

```js
// lib/connect.js 构建的访问器
{
  collection: (collName) => instance.collection(databaseName, collName),  // 默认 db
  db: (dbName) => {
    return {
      collection: (collName) => instance.collection(dbName, collName)     // 任意 db
    };
  }
}
```

关键发现：`this.dbInstance.db(dbName).collection(colName)` 已经支持切换数据库，底层是 `this._adapter.collection(dbName, colName)` — 使用的是**默认连接（default MongoClient）**。

### 多连接池 `ConnectionPoolManager._getPool(name)`

返回指定连接池的 `MongoClient`（已连接）。

### 当前 `lib/mongodb/index.js` `collection()` 方法

接收 `(databaseName, collectionName)` → 用 `this.client.db(dbName)` + 包装成完整 collection 对象（含 runner、缓存、慢查询等）。

---

## 2. `connection` 四种组合行为

| pool | database | 路由路径 |
|:----:|:--------:|---------|
| 无 | 无 | `this.dbInstance.collection(name)` ← 原逻辑，向后兼容 |
| 无 | `logs_db` | 默认连接池 → `this.dbInstance.db('logs_db').collection(name)` |
| `analytics` | 无 | `analytics` 池 → 实例 `databaseName` |
| `analytics` | `reports_db` | `analytics` 池 → `reports_db` |

## 3. 方案设计

### 3.1 `ModelConnection` — 新类型定义

```ts
// types/model/definition.ts
export interface ModelConnection {
  /** 指定连接池名称（须与 pools[].name 对应） */
  pool?: string;
  /** 指定数据库名称（不指定则使用实例初始化时的 databaseName） */
  database?: string;
}
```

加入 `ModelDefinition`:
```ts
interface ModelDefinition<T> {
  // ...existing...
  connection?: ModelConnection;
}
```

### 3.2 `lib/model/index.js` — 验证 `connection` 字段

在 `_validateOptions()` 中增加 `connection` 验证（`connection` 与 `options` 平级，在 `define()` 中单独验证）:

```js
// _validateConnection(connection) — 新增静态方法
static _validateConnection(connection) {
  if (!connection || typeof connection !== 'object') return;

  if (connection.pool !== undefined) {
    if (typeof connection.pool !== 'string' || connection.pool.trim() === '') {
      throw new Error('connection.pool must be a non-empty string');
    }
  }
  if (connection.database !== undefined) {
    if (typeof connection.database !== 'string' || connection.database.trim() === '') {
      throw new Error('connection.database must be a non-empty string');
    }
  }
}
```

`define()` 中在 `_validateOptions()` 之后调用：
```js
if (definition.connection) {
  this._validateConnection(definition.connection);
}
```

### 3.3 `lib/mongodb/index.js` — 新增 `collectionFromClient()`

为支持从**任意外部 MongoClient** 创建 monSQLize 包装 collection，在 MongoDB 适配器上新增方法：

```js
/**
 * 从指定 MongoClient 创建包装 collection（用于多连接池场景）
 * @param {MongoClient} client - 外部 MongoClient 实例
 * @param {string} databaseName - 数据库名称
 * @param {string} collectionName - 集合名称
 * @returns {Object} monSQLize 包装 collection 对象
 * @since v1.2.2
 */
collectionFromClient(client, databaseName, collectionName) {
  // 参数复用现有 collection() 方法的验证
  // 临时替换 this.client，调用 collection()，再恢复
  // ⚠️ 线程安全说明：Node.js 单线程，无竞态风险
  const originalClient = this.client;
  this.client = client;
  try {
    return this.collection(databaseName, collectionName);
  } finally {
    this.client = originalClient;
  }
}
```

> **为什么用临时替换而不是重构**：`collection()` 方法包含了 runner 创建、iid 计算、慢查询注入等复杂逻辑（~100行），重复实现容易出错。临时替换 client 是最小化变更的安全方式，Node.js 单线程保证无竞态。

### 3.4 `lib/index.js` — `model()` 增加路由 + 新增 `_resolveModelCollection()`

在 `model()` 中替换 collection 获取逻辑：

```js
// 原有（不变）：
const collection = this.dbInstance.collection(collectionName);

// 新（有 connection 时路由，无 connection 保持原样）：
const connection = modelDef.definition.connection;
const collection = (connection && (connection.pool || connection.database))
  ? this._resolveModelCollection(collectionName, connection)
  : this.dbInstance.collection(collectionName);
```

新增私有方法：

```js
/**
 * 解析 Model 绑定的 collection
 * @private
 * @param {string} collectionName
 * @param {{ pool?: string, database?: string }} connection
 * @returns {Object} monSQLize 包装 collection
 */
_resolveModelCollection(collectionName, connection) {
  const poolName = connection.pool;
  const dbName = connection.database || this.databaseName;

  if (poolName) {
    // 需要多连接池支持
    if (!this._poolManager) {
      const err = new Error(
        `Model '${collectionName}' requires pool '${poolName}' but no pools are configured. ` +
        `Add 'pools' to MonSQLize constructor options.`
      );
      err.code = 'NO_POOL_MANAGER';
      throw err;
    }

    const client = this._poolManager._getPool(poolName);
    if (!client) {
      const err = new Error(
        `Pool '${poolName}' not found. Available pools: ` +
        `[${this._poolManager.getPoolNames().join(', ')}]`
      );
      err.code = 'POOL_NOT_FOUND';
      throw err;
    }

    // 从指定池 + 数据库创建 collection（使用 adapter 包装）
    return this._adapter.collectionFromClient(client, dbName, collectionName);
  }

  // 只切换数据库，用默认连接
  return this.dbInstance.db(dbName).collection(collectionName);
}
```

### 3.5 缓存策略

`_modelInstances` 当前用 `collectionName` 作为 Map 键。  
由于 `Model.define()` 的 `collectionName` 全局唯一（重复 define 会报错），同一个 `collectionName` 只会有一个 `connection` 配置。**缓存键不需要变更**，现有 `redefine()` 失效机制（`_redefinedNames`）同样适用。

## 6. 风险评估

| 风险 | 缓解 |
|------|------|
| `collectionFromClient()` 临时替换 client 的线程安全 | Node.js 单线程，同步操作，无风险 |
| 多连接池未配置时 pool 引用报错 | 明确 `NO_POOL_MANAGER` 错误码 + 友好提示 |
| `redefine()` 后缓存未失效 | 现有 `_redefinedNames` 机制已覆盖 |
| 指定 pool 的 MongoClient 未连接 | `_poolManager._getPool()` 返回已连接的 client，addPool 时已验证连接 |
